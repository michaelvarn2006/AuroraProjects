# Проект 1: TemperatureConverter (Конвертер температур)

## Описание
Приложение для быстрого пересчёта температуры между шкалами **Цельсия**, **Фаренгейта** и **Кельвина**.  
Разработано для мобильной ОС **Аврора** с использованием нативного стека `C++/QML`.  
Интерактивный интерфейс на базе компонентов **Sailfish Silica** обеспечивает плавную работу и соответствие фирменному стилю платформы.

## Функциональные возможности
- Ввод числового значения температуры через поле `TextField` с цифровой клавиатурой.
- Выбор направления конвертации из трёх вариантов с помощью `ComboBox`:
  - **°C → °F**
  - **°F → °C**
  - **°C → K**
- **Автоматический пересчёт** результата при изменении ввода или выборе направления (без кнопки «Посчитать»).
- Отображение результата с округлением до двух знаков после запятой; при пустом или некорректном вводе выводится `—`.
- Выдвижное меню `PullDownMenu` с пунктом **«Очистить»**, сбрасывающим поле ввода.

## Используемые технологии
- **Языки:** `C++` (только точка входа), `QML`, `JavaScript` (вычисления).
- **Фреймворк:** `Qt 5.6` (системная версия в ОС Аврора 5).
- **UI-библиотека:** `Sailfish.Silica 1.0` (компоненты `Page`, `TextField`, `ComboBox`, `Label`, `PullDownMenu`).
- **Сборка:** `qmake`, Аврора SDK (эмулятор x86_64).

## Архитектура
Приложение построено по классической схеме QML-приложения: корневой `ApplicationWindow` загружает главную страницу `MainPage.qml`.  
Все вычисления реализованы в виде JavaScript-функций внутри страницы, что обеспечивает реактивность через привязки (bindings).  
Отступы и размеры шрифтов задаются через глобальный объект `Theme`, что гарантирует адаптивность на разных экранах.

## Сборка и запуск
1. Откройте проект `TemperatureConverter.pro` в Аврора IDE.
2. Выберите целевой таргет (например, эмулятор x86_64).
3. Нажмите **«Собрать»** и **«Запустить»** — приложение будет развёрнуто в эмуляторе.

# Проект 2: Notes (Заметки)

## Описание
Многостраничное приложение для ведения заметок с сохранением данных между сеансами.  
Реализовано с использованием модели данных на `C++` (`QAbstractListModel`), что демонстрирует бесшовную интеграцию `C++` и `QML`.  
Интерфейс соответствует паттернам навигации ОС Аврора (стек страниц, диалоги, контекстные меню).

## Функциональные возможности
- Главный экран со списком заметок (текст + дата создания).
- Добавление заметки через диалог `Dialog` с многострочным полем `TextArea`; кнопка **«Принять»** активна только при непустом тексте.
- Удаление заметки из контекстного меню (долгое нажатие) с анимацией отмены (`remorseDelete`).
- Сохранение заметок в JSON-файл в каталоге данных приложения (`AppDataLocation`) и автоматическая загрузка при старте.
- Информативная обложка (`Cover`), показывающая количество заметок и позволяющая быстро создать новую или удалить первую.
- Пустое состояние списка (`ViewPlaceholder`) с подсказкой для пользователя.

## Используемые технологии
- **Языки:** `C++` (модель, хранение), `QML` (интерфейс), `JavaScript` (обработка сигналов).
- **Фреймворк:** `Qt 5.6` (`QAbstractListModel`, `QDateTime`, `QJsonDocument`).
- **UI-библиотека:** `Sailfish.Silica 1.0` (`SilicaListView`, `Dialog`, `PullDownMenu`, `ContextMenu`, `Cover`).
- **Сборка:** `qmake`, Аврора SDK.

## Архитектура
- **Модель (`NotesModel`)** — наследник `QAbstractListModel`; хранит список структур `Note` (текст + дата). Предоставляет роли `text` и `createdAt` для QML-делегатов. Методы `addNote()` и `removeNote()` доступны из QML через `Q_INVOKABLE`. Сигнал `countChanged` обновляет счётчик в заголовке.
- **Интеграция:** модель зарегистрирована в QML через `qmlRegisterType` (или контекстное свойство) и используется как `model` для `SilicaListView`.
- **Навигация:** `PageStack` для перехода к диалогу добавления и возврата.
- **Хранение:** сериализация в JSON с помощью `QJsonDocument`; загрузка при создании модели, сохранение после каждого изменения.
- **Обложка:** `CoverBackground` с динамическими надписями и двумя `CoverAction` для быстрых действий.

## Сборка и запуск
1. Импортируйте проект в Аврора IDE.
2. Убедитесь, что в `.pro` файле добавлена опция `CONFIG += auroraapp_i18n` для локализации (если требуется).
3. Соберите проект для эмулятора или реального устройства.
4. При первом запуске каталог данных будет создан автоматически; заметки сохраняются в `notes.json`.

# Проект 3: ImageProcessor (Обработка изображений)

## Описание
Демонстрационное приложение, реализующее граничные вычисления: пользователь выбирает фотографию из галереи, после чего она обрабатывается в фоновом потоке (преобразование в градации серого и вычисление средней яркости).  
Во время обработки интерфейс остаётся отзывчивым благодаря многопоточности.  
Результат отображается через собственный `QQuickImageProvider`, обеспечивающий эффективную передачу `QImage` из `C++` в `QML`.

## Функциональные возможности
- Выбор изображения из галереи устройства через `ImagePickerPage` (модуль `Sailfish.Pickers`).
- Отображение оригинального выбранного изображения.
- Кнопка **«Обработать»** запускает фоновую задачу (активна только при наличии файла и если не идёт обработка).
- Во время обработки отображается индикатор `BusyIndicator`; интерфейс не блокируется.
- После завершения показываются:
  - обработанное изображение (в градациях серого),
  - средняя яркость (0–255, с одним знаком после запятой),
  - время обработки в миллисекундах.
- Страница прокручивается (`SilicaFlickable`).

## Используемые технологии
- **Языки:** `C++` (основная логика, многопоточность, работа с `QImage`), `QML` (интерфейс).
- **Фреймворк:** `Qt 5.6` (`QThread`, `QImage`, `QQuickImageProvider`, `QElapsedTimer`, `QMutex`).
- **UI-библиотека:** `Sailfish.Silica 1.0`, `Sailfish.Pickers 1.0`.
- **Сборка:** `qmake`, Аврора SDK.

## Архитектура
- **Многопоточность:** Класс `ImageProcessor` (главный поток) управляет рабочим потоком `QThread`, в котором живёт объект `ProcessingWorker`. Запрос на обработку отправляется через сигнал `processRequested`, а результат возвращается через сигнал `finished`. Это исключает блокировку GUI.
- **Обработка:** `ProcessingWorker` загружает изображение, конвертирует в `Format_Grayscale8`, сканирует пиксели через `constScanLine()` для вычисления средней яркости, замеряет время.
- **Отображение результата:** Создан `ResultImageProvider`, наследник `QQuickImageProvider`. Он хранит последнее обработанное изображение (с мьютексом) и отдаёт его по запросу. URL результата формируется с суффиксом-версией, который меняется при каждой обработке, что заставляет QML-элемент `Image` перезагружать картинку (свойство `cache: false`).
- **Разрешения:** В `.desktop` файле указано разрешение `Pictures` для доступа к галерее.

## Сборка и запуск
1. Откройте проект в Аврора IDE.
2. Убедитесь, что в `.pro` файле прописана ссылка на библиотеку `auroraapp` (и при необходимости `sailfishsilica-qt5`).
3. Соберите и запустите на эмуляторе. Обратите внимание: в эмуляторе галерея может быть пуста, поэтому для теста можно поместить изображение вручную в каталог `~/Pictures` эмулятора или использовать функцию выбора файлов.

# Общие замечания по всем проектам
- Все приложения разработаны с учётом ограничений **ОС Аврора 5** (`Qt 5.6`, импорты версий 2.0 и 1.0).
- Строки интерфейса обёрнуты в `qsTr()` для поддержки локализации.
- Проекты собираются и тестируются в Аврора SDK (эмулятор x86_64). Для реальных устройств необходимо выбрать соответствующий таргет (`armv7hl` или `aarch64`).
- Соблюдены принципы песочницы: собственные каталоги данных используются без дополнительных разрешений; для доступа к пользовательским файлам (изображения) запрашивается только необходимое разрешение.
